  SYSTEM$CLEANUP_DATABASE_ROLE_GRANTS  ¶

Revokes privileges on dropped objects from the share and grants the database
role to the share.

##  Syntax  ¶

    
    
    SYSTEM$CLEANUP_DATABASE_ROLE_GRANTS( '<database_role_name>' , '<share_name>' )
    

Copy

##  Arguments  ¶

` '  _ database_role_name  _ '  `

    

The name of the database role.

If the identifier is not fully qualified in the form of ` _ db_name  _ .  _
database_role_name  _ ` , the command uses the database role in the current
database for the session.

` '  _ share_name  _ '  `

    

The name of the share.

##  Access control requirements  ¶

To call this function, the active role must have the global [ MANAGE GRANTS
privilege  ](../../user-guide/security-access-control-privileges.html#label-
global-privileges) .

##  Usage notes  ¶

None.

##  Example  ¶

Call the function:

>
>     SELECT SYSTEM$CLEANUP_DATABASE_ROLE_GRANTS('mydb.dbr1' , 'myshare');
>
