  APPLICATION_JSON  ¶

Returns a JSON object that specifies the JSON message to use for a
notification. This is a helper function that you use to construct a message
object for the [ SYSTEM$SEND_SNOWFLAKE_NOTIFICATION  ](../stored-
procedures/system_send_snowflake_notification) stored procedure.

See also:

    

[ Using SYSTEM$SEND_SNOWFLAKE_NOTIFICATION to send notifications
](../../user-guide/notifications/snowflake-notifications) , [
SYSTEM$SEND_SNOWFLAKE_NOTIFICATION  ](../stored-
procedures/system_send_snowflake_notification) , [ TEXT_HTML  ](text_html) , [
TEXT_PLAIN  ](text_plain)

##  Syntax  ¶

    
    
    SNOWFLAKE.NOTIFICATION.APPLICATION_JSON( '<message>' )
    

Copy

##  Arguments  ¶

` '  _ message  _ '  `

    

Content of the message to send.

You do not need to escape the double quotes around strings within the message
(for example, double quotes around the keys and values). The function escapes
these double quotes for you.

##  Returns  ¶

A JSON-formatted string that specifies a message for the [
SYSTEM$SEND_SNOWFLAKE_NOTIFICATION  ](../stored-
procedures/system_send_snowflake_notification) stored procedure to send.

For example, suppose that you call the function and pass in a JSON message:

    
    
    SELECT SNOWFLAKE.NOTIFICATION.APPLICATION_JSON('{"data": "hello world"}');
    

Copy

The function returns the following JSON-formatted string:

    
    
    '{"application/json":"{\"data\": \"hello world\"}"}'
    

Copy

Note how the function escapes the double quotes around the keys and values in
your message.
